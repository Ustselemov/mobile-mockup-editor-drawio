# V5 Scope And Supported Artifacts

## Product re-scope
The product is moving from a mobile-only mockup editor to a universal visual editor with controlled draw.io compatibility.

## Supported artifact families in the current V5 pass
- Mobile wireframes: `PASS`
- Desktop/web wireframes: `PARTIAL`
- Flowchart boards: `PARTIAL`
- Architecture diagrams: `PARTIAL`
- Journey / swimlane boards: `PARTIAL`
- Tree / mind / org diagrams: `PARTIAL`
- Whiteboard note clusters: `PARTIAL`
- Data / ERD-lite diagrams: `PARTIAL`

## Controlled scope
- Keep SVG-first rendering.
- Keep JSON as the in-memory model.
- Keep draw.io support as a controlled subset.
- Do not claim full draw.io or Figma parity.
