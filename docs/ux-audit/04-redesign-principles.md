# Redesign Principles

- Visual-first: every insertable item should have a recognizable preview before insertion.
- Mouse-first: creation, movement, quick actions, and navigation must stay comfortable without command memorization.
- Direct manipulation: selection, parent boundaries, drag feedback, and contextual actions must stay near the object.
- Progressive disclosure: deep settings go right; fast actions stay near object or in the left library.
- Component-first: presets and reusable blocks should beat demo screens as the default entry point.
- Sample screens are secondary: presets remain useful, but not as the main learning surface.
