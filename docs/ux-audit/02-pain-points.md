# Pain Points

## Blocker UX issues
- Old left panel mixed library, templates, and layers, so create flow and structure flow fought each other.
- Sample screens previously looked like the main entry path and distracted from real construction flow.

## High UX issues
- Top bar still has medium reading load for frequent actions.
- Inspector remains functionally rich but visually dense.
- Layers is clearer than before but still not as lightweight as the board flow.

## Medium UX issues
- Quick insert is available only via floating button, not free-area click.
- Favorites/recent currently optimize click insertion more than drag flows.
- Context toolbar is useful, but style presets are not yet exposed there.

## Cosmetic issues
- Some labels are still developer-facing rather than product-facing.
- The shell uses a cleaner hierarchy now, but icon language can still be unified further.
