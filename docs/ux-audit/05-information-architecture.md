# Information Architecture

## Primary order
1. Create / insert
2. Manipulate
3. Inspect / edit
4. Organize
5. Export

## Shell mapping
- Left: create and insert.
- Center: manipulate on canvas.
- Near object: quick actions.
- Right: inspect and organize.
- Top: global document and view actions.

## Decision
Demo screens and weak sample galleries are not the primary product entry. Primary entry is library + components + patterns + quick insert.
