# Current UI Inventory

## Areas
- Top bar: file actions, canvas controls, arrange menu, project menu, code toggle.
- Left sidebar: visual library with `Elements`, `Components`, `Patterns`, `Screens`, `Recent`, `Favorites`.
- Board/canvas: infinite board, screen editing, selection, resize, connect, quick insert trigger, minimap, empty state.
- Right sidebar: `Inspector` and `Layers` secondary tabs.
- Bottom controls: status bar with zoom, grid, validation, selection, pan hint.
- Dialog-like surfaces: arrange dropdown, project dropdown, quick insert popover.
- Palette/library: preview cards, search, favorites, recent, click-to-insert, drag-to-insert for primitive elements.
- Screen preset area: integrated into `Screens` mode and intentionally secondary.
- Example/demo entry points: `Load demo` stays in project menu, no longer primary entry flow.

## Current strengths
- Full-screen shell without page scroll.
- Mouse-first canvas flow is already present.
- Board overlays now support contextual actions and minimap.
- Library is visual-first instead of text-only.

## Remaining weak spots
- Top bar still carries more text than ideal.
- Quick insert is floating-button based only; empty-area invocation is not added yet.
- Inspector still exposes many raw fields instead of richer controls.
